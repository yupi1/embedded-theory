Фронт сигналу

Наростаючий та спадаючий фронт, хто їх генерує, таймінги — інтерактивний довідник для STM32 / ESP32 розробника.

01 — Визначення
⬆ Наростаючий фронт (Rising Edge)

Перехід сигналу зі стану LOW → HIGH. Напруга зростає від 0 до Vcc.

↑ 0 → Vcc POSEDGE
⬇ Спадаючий фронт (Falling Edge)

Перехід сигналу зі стану HIGH → LOW. Напруга падає від Vcc до 0.

↓ Vcc → 0 NEGEDGE
Ключова ідея: реальний фронт — не миттєвий стрибок, а перехідний процес. Його крутизна (slew rate) = dV/dt [В/нс]. Занадто повільний → помилкові спрацювання; занадто швидкий → EMI та відбиття.
02 — Інтерактивна анімація
Slew Rate: 6 нс

📊 Легенда осцилограми

  • DATA — сигнал даних, змінюється відносно фронтів CLK

Підказка: наведіть курсор на хвилю — побачите рівень і час. Слайдер змінює крутизну фронту в реальному часі.
03 — Джерела фронтів
🔧 Тактовий генератор (HSE / HSI / PLL)

Головний генератор фронтів — кварцовий резонатор або RC-осцилятор всередині МК.

  • HSI — внутрішній RC, ±1% точність, ~8/16 МГц
  • HSE — зовнішній кварц, ±20 ppm точність, до 25 МГц
  • PLL — множник частоти: HSE 8 МГц × 9 = 72 МГц (STM32F1)
  • ESP32 — вбудований XTAL 40 МГц + PLL до 240 МГц
GPIO Push-Pull

Обидва напрямки активні. P-МОП тягне до Vcc (RISING), N-МОП — до GND (FALLING). Фронти круті та симетричні.

GPIO Open-Drain

Лише спадаючий фронт активний. RISING — пасивно через pull-up резистор. Повільніше, залежить від R·C.

⚡ Периферія: таймери, UART, SPI, I²C
  • Timer PWM — фронти генеруються апаратно по рахунку ARR/CCR
  • UART TX — старт-біт це завжди спадаючий фронт (HIGH→LOW)
  • SPI CPOL=0 — CLK у спокої LOW, перший фронт — наростаючий
  • I²C — open-drain SCL та SDA, фронти формуються pull-up 4.7 кОм

05 — EXTI: переривання по фронту
STM32 — EXTI
  • EXTI лінії 0–15 відповідають пінам GPIO
  • EXTI_RTSR — Rising Trigger Select
  • EXTI_FTSR — Falling Trigger Select
  • Можна вмикати обидва (BOTH)
  • HAL: GPIO_MODE_IT_RISING
ESP32 — GPIO ISR
  • gpio_set_intr_type(pin, GPIO_INTR_POSEDGE)
  • GPIO_INTR_NEGEDGE — спадаючий фронт
  • GPIO_INTR_ANYEDGE — будь-який фронт
  • ISR: gpio_isr_handler_add()
  • Дебаунс: SW фільтр або RC + компаратор
Важливо: при ANYEDGE переривання спрацьовує двічі на період. Корисно для duty cycle або квадратурного енкодера.
