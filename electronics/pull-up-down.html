Pull-up /
Pull-down

Один резистор — і лінія ніколи не «висить у повітрі». Floating state — найпоширеніша причина глюків на GPIO. Цей конспект пояснює як, чому і коли використовувати підтягуючі резистори.

STM32 ESP32 Arduino GPIO Electronics Fundamentals
01
Навіщо взагалі потрібні?

Цифровий вхід МК читає HIGH або LOW — і більше нічого. Але якщо пін нікуди не підключений (або підключений через кнопку, яка зараз розімкнута), він знаходиться в стані floating — невизначеному стані, де випадковий електромагнітний шум, ємнісні наводки або залишковий заряд дають хаотичний 0 або 1.

⚠ Floating State
Floating — це не «десь між 0 і 1». Це непередбачуваний стан, де значення може спонтанно змінюватись навіть без жодної зовнішньої дії. Особливо небезпечний для входів типу Reset, Enable, CS — де хибний рівень може перезавантажити або вимкнути пристрій.
02
Pull-up резистор

Pull-up — резистор, підключений між піном і VCC. Він «підтягує» лінію до HIGH, коли вона ні з чим не з'єднана.

Логіка роботи:

  • Кнопка відпущена — пін з'єднаний з VCC через R. Читається HIGH. Через резистор струм мінімальний (не КЗ).
  • Кнопка натиснута — пін підключений до GND. Читається LOW. Струм через R обмежений: I = VCC / R.
👆 Ключова думка
Pull-up → активний LOW. Кнопка натиснута — логічний нуль. Саме тому кнопки на платах типу Nucleo, ESP32-DevKit підключені через pull-up: натиснути = LOW.
03
Pull-down резистор

Pull-down — резистор між піном і GND. Утримує лінію в LOW, поки ніхто не подає HIGH.

  • Кнопка відпущена — пін тягнеться до GND через R. Читається LOW.
  • Кнопка натиснута — пін підключений до VCC. Читається HIGH. Струм через R — в GND.
👆 Ключова думка
Pull-down → активний HIGH. Кнопка натиснута — логічна одиниця. Більш «природна» логіка для початківців, але рідше використовується на платах (бо MCU мають вбудовані pull-up, але зазвичай не мають pull-down).

05
Як вибрати номінал резистора?

Це компроміс між захистом від шуму (менший R → сильніше тягне → стабільніше) та споживанням струму (менший R → більший струм через R → більше тепла і витрати батареї).

I = VCC / R Приклад: VCC=3.3V, R=10kΩ → I = 0.33 мА в активному стані
Номінал Струм @ 3.3V Типове застосування
470 Ω — 1 kΩ 3.3 — 7 мА Дуже шумне середовище, захист. Але велике споживання
4.7 kΩ — 10 kΩ 0.33 — 0.7 мА Золотий стандарт. GPIO кнопки, I²C (4.7k), SPI CS
47 kΩ — 100 kΩ < 0.1 мА Low-power пристрої, повільні лінії. Погана стійкість до ємнісних навантажень
⚠ I²C спеціальний випадок
Для I²C номінал pull-up залежить від швидкості шини: Standard Mode (100kHz) — 4.7 kΩ, Fast Mode (400kHz) — 2.2 kΩ, Fast Mode Plus (1MHz) — 1 kΩ. Занадто великий R = надто повільний фронт = помилки при великих швидкостях.
📌 Вбудовані pull-up MCU
Внутрішні pull-up STM32 — зазвичай 20–50 kΩ. Це достатньо для кнопок, але замало для I²C на 400kHz+. Для I²C завжди ставте зовнішні резистори.
06
Внутрішні vs зовнішні

Більшість MCU мають вбудовані підтягуючі резистори, які вмикаються програмно. Це зручно — не треба паяти зайві деталі. Але вони мають обмеження.

Приклади конфігурації:

// STM32 HAL — увімкнення внутрішнього pull-up
GPIO_InitStruct.Mode = GPIO_MODE_INPUT;
GPIO_InitStruct.Pull = GPIO_PULLUP;   // або GPIO_PULLDOWN
HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
// ESP32 Arduino — увімкнення внутрішнього pull-up
pinMode(PIN, INPUT_PULLUP);
// або
pinMode(PIN, INPUT_PULLDOWN); // є у ESP32, немає у звичайному Arduino
// Arduino (ATmega) — pull-up через регістр
PORTB |= (1 << PB0);  // внутрішній pull-up на PB0 в режимі INPUT
// або через Arduino API:
pinMode(2, INPUT_PULLUP);

07
Open-drain і Open-collector

Open-drain (MOSFET) і open-collector (BJT) — це виходи, які можуть тільки тягнути лінію до GND (LOW), але не можуть самостійно подати HIGH. Щоб лінія піднялась до HIGH — потрібен зовнішній pull-up.

Драйвер пише 1
→
Транзистор ВИМК
→
Pull-up тягне лінію HIGH
Драйвер пише 0
→
Транзистор УВІМК
→
Лінія замикається на GND → LOW
💡 Чому I²C використовує open-drain
На I²C шині може бути кілька пристроїв, і кожен може тягнути лінію до LOW (wired-AND). Якщо будь-який тягне LOW — шина LOW. Один pull-up піднімає шину HIGH тільки якщо всі відпустили. Це дозволяє clock stretching і multi-master arbitration без конфлікту виходів.
⚠ Без pull-up — open-drain не працює
Якщо на I²C шині немає pull-up — SDA і SCL ніколи не підніметься до HIGH. Мікроконтролер побачить лише LOW. Забутий pull-up — типова помилка при роботі з I²C на power-up / breadboard.
08
Типові схеми застосування

Де зустрічається на практиці:

  • Кнопка до GND + pull-up — найпоширеніша схема. Кнопка натиснута → LOW. INPUT_PULLUP у Arduino/STM32.
  • I²C шина — обов'язково pull-up 4.7k (100kHz) або 2.2k (400kHz) на SDA і SCL до VCC.
  • Reset пін мікросхеми — зазвичай active-low. Pull-up утримує чіп у нормальному стані; натискання кнопки → GND → reset.
  • UART RX у спокої — pull-up до VCC. Коли нема передачі, лінія стабільно HIGH (idle state протоколу).
  • SPI CS (Chip Select) — pull-up щоб CS=HIGH (деселектовано) поки MCU не ініціалізований.
  • MOSFET gate — pull-down 10k до GND. Щоб при старті MCU (поки GPIO не ініціалізовано) транзистор не відкрився випадково.
  • Enable пін силового IC — pull-down або pull-up залежно від полярності (читайте datasheet: EN або /EN).
  • 1-Wire (DS18B20 тощо) — pull-up 4.7k обов'язково. Також може живити сенсор через паразитне живлення.

Питання на співбесіді
Що таке floating state і чому він небезпечний? ▾
Floating (high-impedance) state — стан піна, коли він не підключений до жодного визначеного потенціалу. Вхідний імпеданс GPIO дуже великий, тому навіть мікровольти від EMI можуть змінити стан. Читання такого піна дає непередбачуваний результат — 0 або 1 хаотично. Небезпечно особливо для Reset, Enable, CS, IRQ піків — хибний сигнал може перезавантажити систему або включити модуль живлення.
Яку типову величину pull-up ставлять на I²C і чому? ▾
4.7 kΩ для Standard Mode (100 kHz) і 2.2 kΩ для Fast Mode (400 kHz). Чим менший R — тим швидший фронт наростання напруги (τ = R × C_bus). Але менший R → більший струм при LOW → більше споживання. Вбудовані pull-up MCU (20–50 kΩ) не підходять для I²C — фронт надто повільний при паразитній ємності шини.
Чому open-drain потребує зовнішнього pull-up? ▾
Open-drain виход має лише один транзистор, який може замкнути лінію на GND (LOW), але не може подати HIGH. Щоб лінія стала HIGH — потрібен зовнішній pull-up резистор до VCC. Саме так влаштовано I²C: кілька пристроїв можуть тягнути шину LOW незалежно (wired-AND), і один pull-up піднімає шину HIGH коли всі відпустили.
Чим відрізняються вбудовані pull-up від зовнішніх? Коли які використовувати? ▾
Вбудовані: 20–50 kΩ, конфігуруються програмно (GPIO_PULLUP / INPUT_PULLUP), можуть вимикатись у Sleep-режимі, зручні для кнопок. Зовнішні: будь-який номінал, завжди активні, потрібні для I²C, коли потрібен менший R для стійкості до шуму або коли MCU іде в Sleep але підтягування має зберігатись. Правило: для кнопок → вбудований, для I²C/1-Wire → зовнішній.
Навіщо pull-down на gate MOSFET? ▾
При старті MCU GPIO піни знаходяться у high-impedance стані поки firmware не ініціалізує їх. Без pull-down gate міг би отримати хибний HIGH від ємнісного наводу або EMI — транзистор відкриється і навантаження увімкнеться ненавмисно. Pull-down 10k утримує gate на GND (LOW) і гарантує, що без явного сигналу MOSFET залишається закритим. Критично для силових кіл.
Як вибрати номінал pull-up/pull-down резистора? ▾
Компроміс між споживанням і стійкістю до шуму:
  • 1–4.7 kΩ: сильне підтягування, низький імпеданс, стійко до шуму, але струм 0.7–3.3 мА при активному рівні
  • 10 kΩ: золотий стандарт для GPIO. Струм 0.33 мА @ 3.3V
  • 47–100 kΩ: low-power режим, повільні лінії без ємнісного навантаження
Для I²C — окрема формула: R ≤ (VCC - V_OL_max) / I_OL_min і обмеження по часу наростання.
У проекті I²C перестав працювати після переходу в Sleep. Чому? ▾
Найімовірніша причина: вбудовані pull-up GPIO вимкнулись разом з GPIO-периферією у Sleep-режимі (особливо актуально для STM32 STOP mode). SDA/SCL повисли у floating стані. Рішення: завжди ставити зовнішні pull-up на I²C шину — вони живляться від VCC незалежно від стану MCU і працюють навіть коли MK спить. Інший варіант: залишити GPIO clock активним у Sleep, але це збільшить споживання.
