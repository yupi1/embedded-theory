Основи сигналів

Сигнал — фундамент будь-якої електронної системи. Без розуміння типів, параметрів і форм сигналів неможливо аналізувати фронти, шуми та поведінку мікроконтролера на рівні залізa.

01 — Що таке сигнал

Сигнал — це фізична величина (напруга, струм), яка змінюється в часі і несе інформацію. В електроніці сигнал — це передусім напруга: ми вимірюємо її в точці схеми відносно GND. Коли мікроконтролер "читає кнопку" або "передає байт по SPI" — він працює з напругою на своїх пінах.

KEY
Ключова ідея: Сигнал = зміна напруги в часі. Все інше — параметри цієї зміни: як швидко, як високо, як часто.
INF
Аналогія: Уяви воду в трубі. Тиск — це напруга, потік — це струм. Сигнал — це коли ти регулюєш кран: відкриваєш і закриваєш. Мікроконтролер — це кран, який відкривається і закривається мільйони разів на секунду.
02 — Типи сигналів

Два фундаментальні типи. Вся різниця — в тому, які значення може приймати напруга.

Аналоговий vs Цифровий сигнал
аналоговий цифровий
◈ Аналоговий
◈ Цифровий

Аналоговий — будь-яке значення між 0В і 3.3В. Цифровий — лише HIGH (3.3В) або LOW (0В).

Аналоговий
Напруга приймає будь-яке неперервне значення в діапазоні. Приклад: мікрофон, термістор, ADC вхід STM32.
Цифровий
Напруга приймає лише два рівні: HIGH або LOW. Приклад: GPIO, UART TX, SPI CLK.
В мікроконтролері
GPIO — цифровий. ADC перетворює аналог у число. DAC — навпаки. PWM — цифровий сигнал, що імітує аналоговий.

04 — Форми сигналів

Кожна форма — своя "математика" і своє застосування в embedded.

Форми сигналів
SQUARE WAVE

Прямокутний сигнал. Два рівні: HIGH і LOW. Ідеальний цифровий сигнал. Застосування: GPIO, UART, SPI, I²C, тактовий сигнал CLK.

ФормаОписДе зустрічається в embedded
Square Wave Симетричний прямокутний сигнал, duty cycle = 50%. GPIO, тактовий CLK, цифрові шини
Sine Wave Плавна синусоїда. Натуральна форма AC-сигналу. Виходи DAC, аудіо, AC живлення, осцилятори
Triangle Wave Лінійний підйом і спад. Проміжне між sin та square. ШІМ (основа), VCO, sweep-генератори
Pulse Короткий імпульс HIGH на фоні тривалого LOW. Duty cycle «lt;50%. IR-пульти, тригери, watchdog reset, TIM one-pulse mode
05 — Логічні рівні

Цифрова схема "не знає" точної напруги — вона знає лише HIGH або LOW. Але реальна напруга варіюється, тому існують пороги та noise margin.

WRN
5В пристрій на 3.3В вхід! Якщо підключити 5В GPIO до STM32 (3.3В), HIGH = 5В, але V_IH max для STM32 = VDD + 0.3В = 3.6В. Результат — пошкодження порту. Завжди перевіряй сумісність рівнів або використовуй level shifter.

07 — Код

Читання логічного рівня GPIO і генерація PWM-сигналу з заданим duty cycle на STM32 HAL.

C · STM32 HAL · GPIO + PWM 
/* ── Читання цифрового сигналу (GPIO) ─────────────────────── */
GPIO_PinState state = HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_0);

if (state == GPIO_PIN_SET) {
    // Логічний HIGH — напруга вище V_IH (~2.0В для 3.3В CMOS)
} else {
    // Логічний LOW — напруга нижче V_IL (~0.8В)
}

/* ── Генерація PWM (duty cycle через CCR) ──────────────────── */
// Period: ARR = 999, PSC = 79 → f_pwm = 80MHz / (80 × 1000) = 1 кГц
HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_1);

// Duty cycle 50%: CCR = (ARR+1) * 0.5 = 500
__HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_1, 500);

// Duty cycle 25%
__HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_1, 250);

/* ── Читання аналогового сигналу (ADC) ─────────────────────── */
// Значення 0-4095 → напруга = (val / 4095) * 3.3В
HAL_ADC_Start(&hadc1);
HAL_ADC_PollForConversion(&hadc1, 100);
uint32_t raw = HAL_ADC_GetValue(&hadc1);
float voltage = (raw / 4095.0f) * 3.3f;  // реальна напруга у вольтах
C · ESP32 ESP-IDF v5.x · GPIO + ADC 
/* ── GPIO читання на ESP32 ──────────────────────────────────── */
gpio_set_direction(GPIO_NUM_4, GPIO_MODE_INPUT);
int level = gpio_get_level(GPIO_NUM_4);
// 0 = LOW, 1 = HIGH

/* ── ADC (аналоговий сигнал) на ESP32 ──────────────────────── */
adc_oneshot_unit_handle_t adc_handle;
adc_oneshot_unit_init_cfg_t init_config = {
    .unit_id = ADC_UNIT_1,
};
adc_oneshot_new_unit(&init_config, &adc_handle);

adc_oneshot_chan_cfg_t chan_config = {
    .atten    = ADC_ATTEN_DB_12,  // діапазон 0-3.3В
    .bitwidth = ADC_BITWIDTH_12,  // 12 біт = 0-4095
};
adc_oneshot_config_channel(adc_handle, ADC_CHANNEL_0, &chan_config);

int raw_value;
adc_oneshot_read(adc_handle, ADC_CHANNEL_0, &raw_value);
// raw_value → 0..4095 відповідає 0..3.3В
Q&A — Питання на співбесіді
Сигнал — це фізична величина (зазвичай напруга), що змінюється в часі і несе інформацію.

Аналоговий: напруга приймає будь-яке неперервне значення в діапазоні (наприклад, 0–3.3В). Приклад: вихід термістора — 1.24В при 25°C, 0.91В при 50°C.

Цифровий: напруга приймає лише два рівні — HIGH (~VDD) або LOW (~0В). Будь-яке проміжне значення — це або шум, або помилка схеми (невизначена зона).
Duty Cycle — відношення часу HIGH до повного періоду, виражене у відсотках.

Формула: DC = t_HIGH / T × 100%

На STM32 duty cycle задається через регістр CCR таймера: DC = CCR / (ARR + 1) × 100%.

Використовується для: управління яскравістю LED, швидкістю мотора (H-bridge), сервоприводами (50Гц, 1–2мс HIGH), імітації аналогового сигналу через RC-фільтр.
Noise Margin — це "запас міцності" між реальним рівнем сигналу і порогом перемикання. Він показує, наскільки великий шум може накластись на сигнал перед тим, як схема зробить помилку.

Для 3.3В CMOS (STM32):
— NM_HIGH = V_OH(min) − V_IH(min) = 3.3В − 2.0В = 1.3В
— NM_LOW = V_IL(max) − V_OL(max) = 0.8В − 0.1В = 0.7В

Чим більший noise margin — тим стабільніша схема в умовах електромагнітних перешкод.
Forbidden Zone (невизначена зона) — діапазон напруги між V_IL_max та V_IH_min. Для 3.3В CMOS це приблизно 0.8В–2.0В.

Якщо сигнал потрапляє туди:
— Транзистор в буфері знаходиться між насиченням і відсіченням
— Вихід стає непередбачуваним: може бути і HIGH, і LOW
— Виникає metastability — схема "завмирає" у невизначеному стані
— Зростає споживання струму (CMOS споживає струм лише під час перемикання)

Рішення: підтягуючі резистори (pull-up/pull-down), драйвери з гістерезисом (Schmitt trigger).
Частота — визначає як ЧАСТО сигнал перемикається. Регулюється через ARR і PSC.

Duty cycle — визначає яку ЧАСТИНУ кожного циклу сигнал знаходиться в HIGH. Регулюється через CCR.

Так, вони незалежні. Приклади:
— 1кГц, 50% — рівний меандр
— 1кГц, 10% — вузький імпульс при тій же частоті
— 50Гц, 5% — сервопривод (1мс HIGH з 20мс периоду)

На STM32: ARR визначає частоту, CCR — duty cycle. Змінюєш CCR → частота не змінюється.
Фаза — зсув сигналу відносно опорної точки у часі, виражений у градусах (0°–360°) або радіанах.

В embedded фаза критична у:
— SPI/I²C: дані мають бути стабільними на момент відповідного фронту CLK (CPOL/CPHA визначають фазу CLK відносно даних)
— PWM complementary: dead time між двома каналами — це фазовий зсув для захисту H-bridge від наскрізного струму
— I²S (аудіо): фаза між BCLK і LRCK визначає формат даних (I²S, Left-Justified, Right-Justified)

Неправильна фаза = схема читає дані в момент перехідного процесу → помилки.
