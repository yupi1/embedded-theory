## Що являє собою цей проєкт

**Embedded Notes** — це статичний сайт на GitHub Pages для конспектів з вбудованих систем, що готуються до технічного інтерв'ю. Сайт замінює Notion-базу: відображає анімовані діаграми, інтерактивний Q&A-акордеон та waveform-перегляди.

Стек: чистий HTML + CSS + JS, без фреймворків, без npm. Кожна сторінка — це самодостатній `.html` файл.

**Дизайн-система (UI-Kit):** темна тема з осцилографною естетикою. Шрифти: JetBrains Mono (код, мітки) + IBM Plex Sans (body, заголовки). Акцент: `#00d084` — зелений термінального типу.

## Акцентний колір — як обирати

На відміну від попередньої версії системи, де кожна категорія мала свій унікальний hex-колір, у новому UI-Kit акцентний колір прив'язується до **семантики сигналу** — це відповідає осцилографній естетиці:

| Тема / категорія    | `--topic-color`       | Колір       | Логіка вибору                    |
|---------------------|-----------------------|-------------|----------------------------------|
| Електроніка         | `var(--rising)`       | `#00d084`   | Зелений = "все ок", стабільний сигнал |
| Протоколи           | `var(--clk)`          | `#4a9eff`   | Синій = тактовий сигнал CLK      |
| Мікроконтролер      | `var(--falling)`      | `#ff6b4a`   | Помаранчевий = переривання, фронт |
| Пам'ять             | `var(--warning)`      | `#f5c842`   | Жовтий = увага, запис/читання    |
| Архітектура         | `var(--clk)`          | `#4a9eff`   | Синій = системний рівень          |
| RTOS                | `var(--warning)`      | `#f5c842`   | Жовтий = часові аспекти          |
| Embedded C++        | `var(--rising)`       | `#00d084`   | Зелений = код, термінал           |

Кожна змінна задається у `:root { --topic-color: … }` на початку файлу.

## Компоненти UI-Kit і коли їх використовувати

UI-Kit містить такі готові компоненти — всі вони вже вбудовані у `TEMPLATE.html`:

**`<code class="ic">REGISTER</code>`** — inline-виділення назви реєстру, функції або значення прямо у тексті. Наприклад: "регістр `ARR` визначає верхню межу лічильника." Використовуй скрізь де з'являється технічна назва.

**`.badge .badge-*`** — маленькі теги-мітки у шапці сторінки. `badge-rising` (зелений) для "learnt", `badge-neutral` (сірий) для платформи або категорії. `badge-pill` робить їх округлими.

**`.callout .callout-*`** — інформаційні блоки зі значком. `callout-important` + іконка `KEY` — для головної ідеї яку треба запам'ятати. `callout-info` + `INF` — для технічних деталей. `callout-warning` + `WRN` — для типових помилок. Не більше одного callout на секцію.

**`.waveform-demo`** — контейнер для будь-якої діаграми: тайм-діаграма сигналів, структурна схема, блок-схема алгоритму. Всередину вставляй `<svg>` або `<canvas>` з анімацією.

**`.kit-table` всередині `.table-wrap`** — для таблиці реєстрів/термінів. Ліва колонка — назва (`class="mono accent"`), права — опис. Використовуй у секції "Ключові терміни".

**`.stat-card` у `.stat-grid`** — числові параметри: частота, швидкість, напруга, розмір буфера. Дає одразу чотири картки в рядок. Видали всю секцію якщо тема концептуальна (без чисел).

**`.code-block`** — блок коду з заголовком, кнопкою "copy" та ручним підсвіченням синтаксису через `span.kw` (ключові слова — синій), `span.fn` (функції — зелений), `span.reg` (реєстри — жовтий), `span.num` (числа — помаранчевий), `span.cm` (коментарі — сірий курсив).

**`.accordion`** — Q&A секція. Клік на питання відкриває відповідь з анімацією. Один `.accordion-item` на одне питання.

## Як створити нову сторінку — покроковий процес

**Крок 1 — скопіювати шаблон** у правильну папку і перейменувати:
```bash
cp TEMPLATE.html protocols/spi.html
```

**Крок 2 — змінити `<title>`:**
```html
<title>SPI · Конспект · Embedded Systems</title>
```

**Крок 3 — встановити `--topic-color`** у блоці `:root` відповідно до таблиці вище.

**Кроки 4–6 — заповнити шапку:** `page-eyebrow` (категорія), `page-title` (назва + акцентне слово), `page-desc` (1-3 речення опис), `page-tags` (набір badge).

**Крок 7 — заповнити секції** у вказаному порядку:

Секція 01 "Ключові параметри" — stat-cards з числовими характеристиками. Видали якщо тема без чисел. Секція 02 "Що це таке" — основне визначення + callout-important з ключовою ідеєю. Секція 03 "Діаграма" — SVG або canvas анімація всередині `.waveform-demo`. Якщо поки немає — залиш placeholder і додай пізніше. Секція 04 "Ключові терміни та реєстри" — таблиця `.kit-table`. Секція 05 "Код" — обов'язково! Мінімум STM32 HAL приклад. Секція 06 "На що звернути увагу" — callout-warning та callout-info для підводних каменів. Секція 07 "Питання на співбесіді" — акордеон мінімум з 3 запитаннями і розгорнутими відповідями.

**Крок 8 — додати картку в `index.html`:** знайди відповідну секцію категорії і додай `<a class="card">`. Видали клас `soon` коли сторінка готова.

## Правила написання контенту

Основний текст — українська. Технічні терміни (назви реєстрів, функцій, протоколів, стандартів) залишаються в оригінальній англійській формі і обгортаються у `<code class="ic">`. Наприклад: "Регістр `CR1` містить біт `PE` для увімкнення периферії."

У code-block підсвічуй токени вручну через span-класи — це робить код читабельнішим і відповідає естетиці IDE. Не пропускай коментарі (`span.cm`) — вони пояснюють логіку прямо в коді.

Анімації в секції "Діаграма" — це пріоритет. Навіть проста SVG тайм-діаграма з `stroke-dasharray` анімацією краща за відсутність діаграми взагалі.

## Критерії "готова сторінка"

Сторінка вважається **done** коли вона має: заповнену секцію "Що це таке" з визначенням і callout-important, таблицю "Ключові терміни" з мінімум 3 рядками, хоча б один приклад коду (STM32 або ESP32) з підсвіченим синтаксисом, секцію Q&A з мінімум 3 питаннями у акордеоні, і картку в `index.html` без класу `soon` і зі статусом `badge-rising` (learnt).
